Duncan’s Lemmata Collator for velut




Testing

Open your browser’s console, then click “Run tests”. The tests confirm that the Lemmata Collator does what I expect.

About

Note: If you’re not me, you’re unlikely to have much use for this Lemmata Collator.

velut is my Latin rhyming dictionary. All the data for this are stored in Json files on my computer. For example, I have a list of lemmata (dictionary headwords), stored as Json objects that include the inflected forms for each lemma. The purpose of the Lemmata Collator is to read from this list of lemmata, transform it into a simple list of words and the lemmata that each word can be a form of, and write that to my hard disk for further processing. I do this by running the collator.js script in Node.

But I also made this webpage, to demonstrate what the Lemmata Collator does. You can input several lemmata (as Json) in the first box, click “Collate lemmata”, and see the output in the second box. The resultant text can be downloaded or copied to the clipboard.

The input for the Lemmata Collator is the output of the Inflector, which itself takes an array of lemmata and adds the inflected forms. The output of the Lemmata Collator meanwhile will be the input of my Word Data Generator.

The velut website does not yet make full use of the inflected forms generated by the Inflector and collated by the Lemmata Collator. Over several years, I manually added forms for every lemma, and these forms are the words that the website displays. (However, in the “Lemma information” section, the website does display inflection-tables that were generated by the Inflector, instead of the manually-added forms.)

I am now confident enough in the output of the Inflector to connect it to the Word Data Generator via this Lemmata Collator… at least for my “development” instance of velut. For the live website, the forms that were in Excel are still being used.

I’m making the Inflector and Lemmata Collator so I can avoid adding inflected forms manually to velut. It’s a lot of work, but it’s better than continuing the way I was.

Input format

The input needs to be Json objects, in the mongoimport format (so the objects are not enclosed with the usual commas and square brackets of a Json array). Each object must have the string field of Lemma and an object field of Forms. The "Forms" object contains nested objects, ultimately containing string arrays. The Lemmata Collator ignores other fields.

The “Load sample” button will give you some examples of my lemma objects. A fuller example is below, giving the entire declension of the adjective absurdus (including enclitics).

The velut Excel file

The velut website uses a MongoDB database, and this page consumes Json data that could go into the MongoDB database. But I privately had a large Excel file for generating and storing the data in velut. This was far from ideal, so I’ve been replacing the Excel file with Json files and JavaScript scripts.

I added all words to Excel manually, including the list of lemmata for each word; it was quite tedious. The Inflector and Lemmata Collator (and indeed the Word Data Generator) allow me to automate the process more.

Another downside of Excel is that it’s difficult to use version-control tools like Git with it.

Testing in the browser

If you’re not me, you won’t have access to all the input data. But you can run some tests yourself in your browser’s console by clicking the “Run tests” button above. These tests run the Lemmata Collator code against some lemmata.

My workflow for managing velut

The Inflector can generate forms for every lemma in my collection, and I have reviewed all the forms. I can see what the velut website would look like when it’s using the full output of the Inflector (and Lemmata Collator). It will be a while until this is live. But once it is live, I will have everything getting derived from my Json data, and the Excel file will be deprecated.

It’s all part of my long-term project of converting my Excel file into websites and webpages that are easier to share and maintain. I’m very much in a transition period of using the Excel file (or data from it, at least) for some things and my newer websites/webpages for others. But the Lemmata Collator is another step in the process. At the moment, the whole velut project is very convoluted; in the future, it won’t be as bad.
